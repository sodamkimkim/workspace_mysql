<1>. 시스템 정의
@ 영화관련정보조회시스템 (관리자용페이지)
	- 통신에서의 서버 vs 클라이언트 개념 ) 클라이언트 개념
	- 클라이언트 단에서 관리자용페이지 vs 고객용페이지) 관리자용 페이지

(참고)
- 관리자용페이지 CRUD범위 >>> 고객용 CRUD범위
- DB테이블과 view할 테이블 개념 분리하자.
- CRUD 개념 넣을 것.
- 꼭 FK 안써도 됨 , 디테일 보다는 '기능 완성'에 집중

<2>. 시스템 기능 요구사항
(참고)
- 시스템이 수행할 기능명세
- 일단 DB다 설계하고 일부만 활용하면 됨.

@ 정보 생성하기(C)
- 계획대로 textfield써서.
- 권한을 위해 관리자 로그인정보가 필요할 것 같긴함(보류)
@ 정보조회하기(R)
	1. 키워드로 조회) 키워드 --> 영화이름, 개봉연도, 역할이름, 배우이름, 감독이름, 장르(공포, 로맨스, 액션,.. )
	2. 영화상세정보 조회)영화 정보 -> 상세정보 클릭 -> 상세정보) 영화이름, 개봉연월, 관객수, 평점, 줄거리
	3. 배우, 감독상세정보 조회)배우 정보 -> 상세정보 클릭 -> 상세정보) 사람이름, 대표작, 대표역할, 프로필(출생년도, 키, 몸무게, 배우자)
	4. Box-office조회(인기순)
		- 최근 3개월 top1~
		- 최근 1년 top1~
		- 연도별 top1~
	5. 장르별 영화 조회(인기순)
- 권한 최하위도 조회가능(보류)
@ 정보 수정하기(U)
- R의 키워드로 조회 => 수정
- 권한을 위해 관리자 로그인정보가 필요할 것 같긴함(보류)
@ 정보 삭제하기(D)
- 전부 다 조인한 값 조회 => 삭제
- 권한을 위해 관리자 로그인정보가 필요할 것 같긴함(보류)
